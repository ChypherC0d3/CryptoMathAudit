3. ACTIVE ISM USAGE STATUS
================================================================================

CRITICAL FINDING: No evidence that Hyperlane's own mainnet configurations
use WeightedMultisigIsm as the default or routing ISM for any chain.

Evidence:
  a) The mainnet3 TypeScript infra config (*.ts files) contains ZERO references
     to IsmType.WEIGHTED_MERKLE_ROOT_MULTISIG or IsmType.WEIGHTED_MESSAGE_ID_MULTISIG
  b) WeightedMultisig ISM types are ONLY referenced in the test environment config:
     typescript/infra/config/environments/test/multisigIsm.ts
  c) The rust mainnet_config.json lists interchainSecurityModule addresses for
     each chain, but these point to regular MultisigIsm or routing ISM instances,
     not weighted variants

HOWEVER: Third-party applications can deploy their own WeightedMultisigIsm
instances using the deployed factories. Hyperlane's architecture allows any
app to specify a custom ISM. We cannot determine from the repo alone whether
third-party warp routes or dApps use weighted multisig ISMs.

5. THE VALIDATOR MATCHING BOUND BUG (from our recon)
================================================================================

Our previously identified bug (Section 4.2 of hyperlane_recon.txt):
  - _validatorCount = min(validators.length, signatureCount)
  - Validators beyond index signatureCount are never checked
  - Valid quorums can be rejected if matching validators are at high indices

This bug is SEPARATE from the Chainlight fix. The Chainlight fix addressed
duplicate signature replay. Our bug addresses the loop bound limiting which
validators can be matched.

IMPACT ASSESSMENT FOR BUG BOUNTY:
  - If no WeightedMultisigIsm instances are actively securing value on mainnet,
    this is a code-level bug but not exploitable in production
  - The bug would be classified as DoS (false rejection of valid signatures),
    not a security bypass (false acceptance of invalid signatures)
  - Severity would be LOW if no deployed instances exist, MEDIUM if they do
  - Third-party usage is unknown and could elevate severity
