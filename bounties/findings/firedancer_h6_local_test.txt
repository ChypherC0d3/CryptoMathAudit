3. OPTIONS FOR FULL AVX512 DIFFERENTIAL TEST
=========================================================================

OPTION 1: Intel SDE (FREE, LOCAL) -- RECOMMENDED
-------------------------------------------------
Cost: $0.00
Time: ~30 minutes setup, ~10-60 min to run (SDE is slow, ~10-100x)

Intel SDE v10.8.0 (2026-03-15) is free under Intel Simplified License.
Download: https://www.intel.com/content/www/us/en/download/684897/intel-software-development-emulator.html
File: sde-external-10.8.0-2026-03-15-lin.tar.xz (31.9 MB)

Setup steps in WSL2:
  1. Download and extract SDE:
     cd /opt
     wget https://downloadmirror.intel.com/843543/sde-external-10.8.0-2026-03-15-lin.tar.xz
     tar xf sde-external-10.8.0-2026-03-15-lin.tar.xz
     export PATH=/opt/sde-external-10.8.0-2026-03-15-lin:$PATH

  2. Disable yama (required for SDE on Ubuntu):
     echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope

  3. Run AVX512 binary under SDE:
     sde64 -skl -- ./build/linux/gcc/icelake/unit-test/test_ed25519

     Or with our custom test script:
     cd /path/to/h6_test
     ./test_h6_avx512.sh --sde

Limitations:
  - SDE is a FUNCTIONAL emulator (correctness only, not performance)
  - Roughly 10-100x slower than native execution
  - The fuzzer with 10M iterations may take hours under SDE
  - Perfect for differential testing (comparing outputs, not speed)

------------------------------------------------
Cost: $0.00

GitHub Actions runners SOMETIMES have AVX512 (depends on which

4. RECOMMENDED APPROACH (CHEAPEST PATH)
=========================================================================

Phase 1: Local reference testing (FREE, do this NOW)
  - Already done. Reference build works in WSL2.
  - Run our H6 test vectors against reference implementation.
  - This catches bugs in the reference code path.

Phase 2: Intel SDE for AVX512 (FREE, do this next)
  - Download Intel SDE into WSL2 (32 MB download).
  - Run the AVX512-compiled binary under SDE.
  - Compare outputs between reference and AVX512 builds.
  - This catches divergences between the two implementations.
  - Use reduced fuzz iterations (e.g., 100K instead of 10M) to
    compensate for SDE slowness.

Phase 3: Native AVX512 validation (only if needed, ~$0.05)
  - If SDE finds a divergence, confirm on real AVX512 hardware.
  - Spin up a c5.xlarge spot instance for 15-30 minutes.
  - Run full test suite natively at full speed.

TOTAL COST: $0.00 for Phases 1-2, ~$0.05 max for Phase 3.

7. KEY FIREDANCER BUILD SYSTEM NOTES
=========================================================================

- GNUmakefile at repo root, not Makefile
- MACHINE= selects target config from config/machine/*.mk
- MACHINE=linux_gcc_minimal: no NUMA, no threads, POSIX-only (WSL-safe)
- MACHINE=linux_gcc_x86_64: AVX2, has NUMA (fails in WSL2)
- MACHINE=linux_gcc_icelake: AVX512, has NUMA (compiles but crashes
  with SIGILL on non-AVX512 CPUs; use SDE to run)
- Build outputs go to build/<machine>/unit-test/
- Multiple MACHINE targets can coexist simultaneously
- Firedancer source cloned to: /tmp/firedancer (WSL2)
  (shallow clone, ~4081 files)
